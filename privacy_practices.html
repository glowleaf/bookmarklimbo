Bookmark Limbo - Privacy Practices

Single Purpose Description

Bookmark Limbo helps users manage their bookmarks by showing a curated selection of bookmarks that need attention, with AI-generated summaries to help make decisions about them. The extension displays three bookmarks at a time (one old, one medium-aged, and one new) to help users systematically work through their bookmark backlog.

Permission Justifications

Bookmarks Permission

This extension requires access to bookmarks to:

  • Display bookmarks in the new tab page
  • Track which bookmarks have been visited
  • Calculate how long bookmarks have been in limbo
  • Enable bookmark management features
  • Sort and categorize bookmarks by age

Host Permissions

The extension needs access to all URLs to:

  • Generate AI summaries of bookmarked content
  • Enable social media integrations (Twitter, Facebook, Reddit, etc.)
  • Support Telegram message viewing
  • Process bookmark content for display
  • Fetch metadata for bookmarked pages

Remote Code

The extension uses remote code for:

  • OpenAI API calls to generate content summaries
  • Social media API integrations (Twitter, Facebook, Reddit, etc.)
  • Telegram API integration
  • Content processing and analysis

Note: All remote code is from trusted, official APIs and is necessary for core functionality. No third-party scripts are loaded without user consent.

Storage Permission

The extension uses storage to:

  • Save user preferences and settings
  • Track visited bookmarks
  • Store API keys and tokens for integrations
  • Maintain limbo duration settings
  • Cache AI summaries for performance

Note: All data is stored locally using Chrome's storage API. No data is sent to external servers except for necessary API calls.

Tabs Permission

The extension needs access to tabs to:

  • Open bookmarks in new tabs when clicked
  • Track which bookmarks have been visited
  • Enable the new tab override functionality
  • Manage bookmark navigation

Data Usage Compliance Certification

I certify that Bookmark Limbo:

  • Only collects data necessary for core functionality
  • Stores all data locally on the user's device
  • Does not share data with third parties except for necessary API calls
  • Provides clear privacy controls to users
  • Complies with Chrome Web Store's Developer Program Policies
  • Implements appropriate security measures for stored data
  • Provides clear documentation of data usage
  • Respects user privacy and data ownership

Data Collection and Usage

The extension collects and uses data in the following ways:

  • Bookmark Data: Used only for display and management within the extension
  • Settings Data: Stored locally to maintain user preferences
  • API Keys: Used only for the specified integrations and stored securely
  • Usage Data: Tracks visited bookmarks to improve the limbo system

User Control and Transparency

Users have full control over:

  • Which integrations to enable/disable
  • What data is stored locally
  • When to clear stored data
  • How their bookmarks are displayed and managed
